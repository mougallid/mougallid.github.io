version 1

Versionner en local

  • git init:Initialiser le depôt (se mettre sur le bon dossier), mieux a faire depuis Github.com
  • git add . : ajouter tous le modification (le . symbolise tout)
  • git commit -m "explication" : Créer un nouveau commit. git add posse le fichier en zonne d'index. git commit les sauvegarde réellement dans un nouveau commit.

Gérer les commit

  • git log :list de commit
  • git log -n2 . : Afficher les 2 derniers commits
  • git show shaw-1 : voir commit spécifique(cliquer molette souris pour coller)
  • git checkout shaw-1 : Remetre la version de shaw-1
  • git show shaw-1 : Remettre version la plus récente.

Commiter son projet sur Github

  • git add .: Pour ajouté tous les modification.
  • git commit -m "Explication": Créer un nouveau commit
  • git push -u origin : Pousser les commits vers erveur.
